Issues & How to improve

"ปัญหาและวิธีปรับปรุง/พัฒนา"

Issue How to improve
SA ลาออก2คน ก่อนโปรเจ็คจะเสร็จ ซึ่งก่อนจะออก SA ทิ้งงานเลย ไม่เข้าประชุม ไม่เปิดการ์ด ไม่อะไรทั้งสิ้น ทำให้ไม่รู้ว่า Status ตอนนี้ทำอะไรไปบ้าง นัด Meeting ทีมเองทั้งหมดที่ดูโปรเจคนี้ ทดสอบให้ดูไปพร้อมๆกันว่าแต่ละ Step เกิดปัญหาอะไร หากเจอ Issue เปิดการ์ดให้ Dev แก้เอง หากเป็นเรื่องการส่ง API หลังบ้านที่ต้องให้คำตอบ Dev แบบที่เราไม่สามารถตัดสินใจเองได้ จะประสานงานขอความช่วยเหลือจาก PM หรือ SA คนอื่นๆว่าต้องแก้ไปในทิศทางไหน *โปรเจ็คนี้ถือว่าเป็น Maim หลักก็ว่าได้ แต่สุดท้ายระบบก็ถูกนำขึ้นได้ตรงกำหนด
คนในทีมความรู้ไม่มากพอโดยเฉพาะฝั่ง True ทำให้ระหว่างทางเกิดปัญหาเยอะระหว่างทดสอบ การส่งค่าต่างๆส่งไปหลายที่ และทำคนละแบบกับ Dtac ติดปัญหาตรงไหนก็พิมพ์แจ้งในกลุ่มที่มีทีม SA , BA , PM ให้คอยช่วยเหลือแนะนำ ว่าต้องแจ้งใครต่อ และให้ช่วยติดตามเรื่องอื่นๆที่ติดปัญหาให้ไวขึ้น
Data Test True น้อยไม่เพียงพอ จะสามารถมีสิทธิเตรียม Data Test ได้แค่2คนในทีม และ Process การเตรียมค่อนข้างหลายขั้นตอน เมื่อรู้ว่าจะได้ถืองานอะไร จะรีบแจ้งขอData Test ไว้แต่เนิ่นๆก่อนเพราะต้องใช้เวลารออีก กับคอยรายงานผลตลอดว่าถ้ากรณีที่ยังไม่ได้ทดสอบเพราะรอ Data Test
SA เอาการ์ดเข้าบ้านช้า จะไม่รอ SA เอาการ์ดเข้าบ้านให้ครบแล้วค่อยเริ่มงาน แต่จะพยายามไปอ่านเอกสารReq ของโปรเจ็ครอก่อน เพื่อทำความเข้าใจล่วงหน้าและเขียน Test case รอ หาก SA เอาการ์ดเข้าบ้านไม่ครบจากสิ่งที่เราเขียน หรือไม่ตรงกับสิ่งที่เราเขียนจะรีบสอบถามเพื่อรีบอัปเดต Testcase
ระบบหลังบ้านชอบพังบ่อย ทำให้เวลาทดสอบแล้วติดปัญหา หากติดปัญหาจะรีบแคปหน้าจอและ save harfile ให้ทีมตรวจสอบ และให้ทีมที่เกี่ยวข้องช่วยติดตาม
งานเยอะมาก ถืองานหลายสิ่งหลายอย่างพร้อมๆกัน อย่างแรกทำใจให้สบาย เรียงลำดับความสำคัญของงาน ประเมิณงานว่าสามารถทดสอบได้ทันเวลาที่กำหนดมั้ย ถ้าประเมิณแล้วไม่ทัน จะรีบแจ้ง SA ไว้แต่เนิ่นๆเพื่อหาวิธีแก้ไข
ปัญหาสุขภาพ และจิตใจที่เหนื่อยล้าจากการทำงาน พยายามทำงานในเวลางานให้เต็มที่ที่สุด ไม่หักโหมนอกเวลามากเกินไปถ้าไม่จำเป็น พยายามมองในแง่บวกว่าสิ่งที่เราทำได้ประโยชน์อะไรกับเรา ให้มีกำลังใจทำงานมากขึ้น เช่น ได้เรียนรู้มากขึ้น ได้ประสบการณ์มากขึ้น ได้เก่งขึ้น
